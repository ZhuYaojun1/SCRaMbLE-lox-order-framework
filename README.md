# SCRaMbLE lox-order framework

This repository contains the audited, segment-level stochastic SCRaMbLE model and the compact source-data package needed to inspect the reported results. The framework represents each synthetic chromosome as an ordered series of segment and lox copy instances, applies deletion, inversion, and duplication to the current topology, and evaluates an Essentiality Gate after each accepted event.

The repository is a model and source-data release. It is not a calibrated wet-lab induction protocol, a complete biological viability model, or a base-pair-level breakpoint predictor. `p_event` is a per-trajectory, per-step model probability.

## Audited release snapshot

- 488 active loxPsym records: SynII 273, SynIII 100, SynVI 71, and SynIXR 44.
- 604 chromosome-specific blank padding cells excluded.
- 45,509 candidate intrachromosomal lox pairs.
- 36,046 direct positive SynII Hi-C pairs and 9,463 distance-fallback pairs.
- 487 of 488 lox coordinates resolved from sequence motifs.
- 717 exact-mapped ORFs eligible for gene-level analysis on SynII, SynIII, and SynVI.
- 99 ORFs in the strict Essentiality Gate.
- 88 main-grid scenarios, each initialized with 10,000 trajectories and 20 sequential event opportunities.
- Rarefaction depth 1,144 with 100 without-replacement replicates.
- 438,708 Gate-failing trajectories audited by deterministic state replay.
- 196,246 Gate-failing trajectories contained an inversion or duplication before the terminal deletion.
- 328,098 non-deletion precursor events were replayed with zero topology, copy-number, or Gate-status mismatches.

The Hi-C model is named **partially Hi-C-informed sampling with distance-based fallback** because direct contact values are available only for SynII.

## Repository structure

```text
scripts/revised_model/                 simulation, analysis, and audit code
data/processed/                        compact processed dependencies
outputs/revised_model/inputs/          audited model-ready inputs
outputs/revised_model/input_sources/   accessioned GenBank annotations
outputs/revised_model/main_run/        compact main-grid source data
outputs/revised_model/analysis/        diversity, backtesting, null, and risk tables
outputs/revised_model/robustness/      seed and Essentiality Gate summaries
outputs/revised_model/cascade_analysis/ rearrangement-cascade replay data and manuscript assets
outputs/revised_model/reports/         automated checks and revision audit
figures/                               audited PNG and PDF manuscript figures
```

Large original per-scenario event logs, SQLite catalogs, full trajectory records, invalidated legacy outputs, the full manuscript, and raw SRA/GEO/FASTQ data are intentionally excluded. Compact compressed endpoint counts, lox-frequency data, and the derived cascade-replay detail are retained because they support the reported diversity, backtesting, and rearrangement-history analyses. A standalone Section 3.8 figure/table package is included; it is not the full manuscript.

## Installation

Python 3.11 or later is recommended.

```bash
python -m venv .venv
python -m pip install -r requirements.txt
```

## Verification

Run the topology-engine tests:

```bash
python -m unittest discover scripts/revised_model/tests
```

Inspect the recorded automated checks:

```text
outputs/revised_model/reports/automated_consistency_checks.csv
outputs/revised_model/reports/FINAL_REVISION_REPORT.md
```

Verify file integrity against `MANIFEST_SHA256.csv` using a SHA-256 utility.

## Re-running the model

The tracked model-ready inputs permit the simulation to run without downloading sequencing data. A small test run can be written to a separate output directory:

```bash
python scripts/revised_model/run_revised_simulation.py \
  --project-root . \
  --output-root outputs/revised_model \
  --n-trajectories 100 \
  --n-steps 3 \
  --model-key linear_distance \
  --alpha 2.0 \
  --p-event 0.10 \
  --gate-name strict \
  --run-subdir public_test_run \
  --reuse-gate-files
```

Run the 88-scenario formal grid only when sufficient compute and storage are available:

```bash
python scripts/revised_model/run_parallel_formal.py \
  --project-root . \
  --output-root outputs/revised_model
```

The compact public package does not contain the large scenario-level trajectory files and endpoint databases. Those files are regenerated by the formal run and are required to recompute the exact-coordinate gene-risk table and endpoint-isomorphism audit from first principles. Their audited summary outputs are included.

The rearrangement-cascade analysis requires regenerated formal-run event logs as input. Its tracked derived outputs can be inspected directly, and the manuscript assets can be rebuilt from those outputs:

```bash
python scripts/revised_model/build_cascade_manuscript_assets.py
```

See `outputs/revised_model/cascade_analysis/README.md` for the replay boundary, file inventory, validation checks, and interpretation limits.

## Third-party source boundary

The Zhang 2022 supplementary workbook and the Zhou 2023 GEO FASTA are not duplicated in this repository. Their filenames, source identifiers, and SHA-256 hashes are recorded in `outputs/revised_model/inputs/input_provenance.json`. Accessioned GenBank annotation files used for exact coordinate mapping are included under `outputs/revised_model/input_sources/`.

See `DATA_AVAILABILITY.md` for source mapping and `DATA_DICTIONARY.md` for the principal files and fields.
